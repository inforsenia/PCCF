## UP01: INTRODUCCIÓN AL DIN
### 1. Identificación
| Campo | Detalle |
| :--- | :--- |
| **Código** | UP01 |
| **Módulo** | Desarrollo de interfaces (0488) |
| **Duración** | **18 Horas** |
| **Temporalización** | Del **08/09/2025** al **20/09/2025** |

### 2. Fundamentación
**Resultado de Aprendizaje**
*   **RA01.** Genera interfaces gráficos de usuario mediante editores visuales utilizando las funcionalidades del editor y adaptando el código generado.

**Criterios de Evaluación**
*   **a)** Se han utilizado las herramientas del editor visual para crear interfaces de usuario.
*   **b)** Se han empleado los asistentes del editor para generar código de forma automática.
*   **c)** Se ha adaptado el código generado por las herramientas a los requisitos del proyecto.
*   **d)** Se han identificado los distintos elementos de la interfaz y sus propiedades.
*   **e)** Se ha verificado el funcionamiento de la interfaz generada.
*   **f)** Se han documentado los pasos seguidos en el proceso de creación.

**Competencias**
*   **Profesionales:** d), h).
*   **Ocupación:** t), w).

### 3. Organización
**Contenidos**
*   Conceptos de IHC: interfaz de usuario y experiencia de usuario (UX).
*   Tipos de interfaces gráficas: escritorio, web y móvil.
*   Editores visuales: Android Studio.
*   Creación de proyectos: estructura de carpetas y archivos.
*   Paleta de componentes y propiedades del editor.
*   Generación de código: relación entre diseño visual y código.
*   Adaptación y personalización del código generado.

**Metodología**
*   Uso de metodologías activas centradas en el estudiante mediante el **proyecto AppCraft**. El alumnado actúa como equipo de desarrollo que recibe su primer encargo: construir la pantalla de bienvenida de una app real.

**Secuencia (Fases)**
*   **E1:** Exploración del entorno: Android Studio, estructura de proyecto y primer Layout.
*   **E2:** Paleta de componentes: añadir, mover y configurar vistas en el editor visual.
*   **E3:** Del visual al código: analizar y adaptar el XML generado automáticamente.
*   **E4:** Entrega: pantalla de bienvenida funcional con documentación del proceso.

**Recursos**
*   Android Studio, JDK 17+, emulador Android, fichas de prácticas guiadas, documentación oficial de Android Developers.

## UP04: INTERFACES EN ANDROID CON JETPACK COMPOSE
### 1. Identificación
| Campo | Detalle |
| :--- | :--- |
| **Código** | UP04 |
| **Módulo** | Desarrollo de interfaces (0488) |
| **Duración** | **40 Horas** |
| **Temporalización** | Del **23/10/2025** al **20/11/2025** |

### 2. Fundamentación
**Resultado de Aprendizaje**
*   **RA04.** Diseña interfaces gráficas identificando y aplicando criterios de usabilidad y accesibilidad.

**Criterios de Evaluación**
*   **a)** Se han aplicado criterios de usabilidad en el diseño de la interfaz.
*   **b)** Se han implementado interfaces accesibles siguiendo las pautas WCAG.
*   **c)** Se ha utilizado Jetpack Compose para crear componentes de interfaz.
*   **d)** Se ha gestionado el estado de la interfaz de forma reactiva.
*   **e)** Se ha implementado la navegación entre pantallas con Navigation Compose.
*   **f)** Se han realizado pruebas de usabilidad y accesibilidad sobre la interfaz desarrollada.

**Competencias**
*   **Profesionales:** h), j).
*   **Ocupación:** t), w).

### 3. Organización
**Contenidos**
*   Introducción a Jetpack Compose: paradigma declarativo vs imperativo.
*   Composables básicos: Text, Button, Image, TextField.
*   Layouts en Compose: Column, Row, Box, LazyColumn.
*   Estado en Compose: remember, mutableStateOf, state hoisting.
*   Efectos secundarios: LaunchedEffect, SideEffect.
*   Navigation Compose: rutas y paso de argumentos.
*   Criterios de usabilidad: legibilidad, jerarquía visual, retroalimentación.
*   Accesibilidad: semántica, TalkBack, tamaños mínimos de toque.
*   Theming: Material3, esquemas de color y tipografía.

**Metodología**
*   Metodologías activas (AppCraft). El equipo implementa en Compose la app cuyo prototipo diseñó en UP02, aplicando desde el primer día criterios de usabilidad y accesibilidad.

**Secuencia (Fases)**
*   **E1:** Paradigma declarativo: primer composable y previsualización en Android Studio.
*   **E2:** Layouts y composables básicos: construir las pantallas principales de la app.
*   **E3:** Estado y reactividad: formularios, listas dinámicas y gestión del estado.
*   **E4:** Navigation Compose: flujo completo de navegación entre pantallas.
*   **E5:** Theming y accesibilidad: Material3, TalkBack y revisión con Accessibility Scanner.
*   **E6:** Entrega: app completa con navegación, theming y auditoría de accesibilidad.

**Recursos**
*   Android Studio Hedgehog+, Kotlin 1.9+, dispositivo o emulador Android, Accessibility Scanner, documentación Jetpack Compose, fichas de prácticas.
